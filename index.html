Http Server

A standard HTTP server, centered on the concept of routes.

Download this project as a .zip file Download this project as a tar.gz file

Want to create a simple HTTP server but don't wanna mess with figuring out how to perform a connection? How to add routes? How to create such routes?

Just wanna focus on the feature you provide through this server and now how you built it?

This library is for you.

With a flexible and rich API created thanks to the OOP library, you will just have to pass configuration object to the server, along with a list of routes, and just ask it to start!

There are multiple ways to create routes, with some standard behaviors implemented. One of the most important one is an RPC route: it is able to handle RPC, and uses JSON for that.

Installation

npm install git+https://github.com/ymeine/http-server

Usage

Example in LiveScript: 

require! {
    server: 'http-server'
}

{run} = server

options = {
    port: 8080 # default port, use another one if busy
    -log # we don't use a logger
}

routes = [
    # Standard simple routes
    'ping'
    'shutdown'

    # Standard complex routes: here RPC
    {
        type: 'rpc'
        modules: {
             'mod1': require('path1')
             'mod2': {
                 someMethod2: -> it
             }
        }
    }
]

run routes, options

A server on port 8080 will run and respond to routes:

  • GET /ping returns 200
  • GET /shutdown shuts the server down
  • POST /rpc with {"module": "mod1", method: "someMethod"} returns the result of the method
  • POST /rpc with {"module": "mod2", method: "someMethod2", argument: "arg"} returns "arg"
